## 快速阅读路线

如果你是第一次阅读，建议按下面顺序查看：

1. 先阅读本 README，了解整体流程、适用条件和风险提示。
2. 如果是真机环境，先确认手机是否支持解锁 BL，并参考早期教程完成 Magisk Root。
3. 如果是模拟器环境，直接查看新版教程中的“模拟器刷跑”部分。
4. 完成基础环境后，再根据新版教程配置 FakeLocation、权限、隐藏应用列表和路线模拟。

## 适用条件

真机方案通常需要满足以下条件：

- 手机支持解锁 BL。
- 已刷入可用的 Magisk Root，而不是功能受限的官方 Root。
- 能正常安装并授权 FakeLocation、隐藏应用列表、LSPosed/EdXposed 等工具。
- 定位权限、后台权限和系统定位设置可以按教程调整。

部分机型不适合直接参考本教程，例如没有稳定 BL 解锁方案的机型，或系统对 Root、定位模拟限制较强的机型。模拟器方案虽然门槛较低，但同样存在软件兼容和账号异常风险。

## 核心流程概览

### 真机方案

真机方案的核心思路是：先获取稳定 Root 环境，再通过 FakeLocation 模拟运动路线。

大致流程如下：

1. 解锁 BL，并刷入 Magisk。
2. 安装运动世界校园、FakeLocation、隐藏应用列表和相关框架工具。
3. 按教程调整运动世界校园的定位权限，关闭可能影响定位精度的系统设置。
4. 配置隐藏应用列表，减少 Root、框架、定位工具被目标应用识别的概率。
5. 在 FakeLocation 中绘制操场路线，开启路线模拟后，再进入运动软件开始记录。

### 模拟器方案

模拟器方案适合没有合适真机，或想在电脑上完成环境配置的情况。

大致流程如下：

1. 安装安卓模拟器，例如夜神、雷电等。
2. 将运动世界校园、FakeLocation 等 APK 安装到模拟器中。
3. 在模拟器中刷入 Magisk 或对应框架环境。
4. 按真机方案的思路配置定位模拟、应用权限和路线模拟。
5. 启动运动软件并检查定位、轨迹和记录状态是否正常。

## 常见问题

### 1. 为什么教程里强调 BL 和 Magisk？

FakeLocation 的部分功能依赖更高权限。没有稳定 Root 环境时，路线模拟可能无法正常生效，也可能被系统或应用限制。

### 2. 为什么定位没有变化？

常见原因包括 FakeLocation 没有 Root 权限、系统定位设置不正确、运动软件没有定位权限、路线模拟没有启动，或模拟位置被系统限制。建议按教程逐项检查权限和定位设置。

### 3. 模拟器会不会更安全？

不一定。模拟器更方便配置和多开，但它本身也可能被应用识别。是否使用模拟器，需要根据软件版本、模拟器版本和实际检测结果判断。

### 4. 下载链接失效怎么办？

原教程中的部分外部下载链接可能会失效。建议优先从软件官方渠道、可信社区或原项目发布页获取对应版本，并注意校验来源。
